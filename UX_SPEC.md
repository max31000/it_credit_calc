## 1. Общая архитектура страницы

### 1.1 Тип приложения

SPA (Single Page Application) без роутинга. Вся логика — на одной странице. Прокрутка — вертикальная. Нет модальных окон, нет навигации между страницами.

### 1.2 Структура DOM (сверху вниз)

```
<body>
  <Header />                          // фиксированная высота, не sticky
  <StickyMetricsBar />                // sticky при скролле
  <main>
    <Section id="params">             // Секция 2: параметры ипотеки
    <Section id="slip">               // Секция 3: сценарий слёта
    <Section id="comparison">         // Секция 4: сравнение стратегий
    <Section id="charts">             // Секции 5–7: графики
    <Section id="advice">             // Секция 8: советы
  </main>
  <Footer />                          // минимальный
</body>
```

### 1.3 Контейнер

Максимальная ширина контента: `1280px`. Центрирование по горизонтали. Горизонтальные отступы:
- Desktop: `px: 40px`
- Tablet: `px: 24px`
- Mobile: `px: 16px`

### 1.4 Вертикальные отступы между секциями

`gap: 24px` между секциями на desktop и tablet, `gap: 16px` на mobile.

## 2. Дизайн-токены и цветовая система

### 2.1 Цветовая палитра

| Токен | Hex | Применение |
|---|---|---|
| `--color-primary` | `#228be6` | Стратегия Б, основные акценты, кнопки |
| `--color-strategy-a` | `#f76707` | Стратегия А (оранжевый) |
| `--color-strategy-b` | `#228be6` | Стратегия Б (синий) |
| `--color-slip` | `#e03131` | Линия слёта, опасные значения |
| `--color-safety` | `#2f9e44` | Точка безопасности, позитивные значения |
| `--color-purple` | `#7048e8` | Вертикальная линия safetyPoint на графике 3 |
| `--color-warning-border` | `#fd7e14` | Рамка секции слёта |
| `--color-winner-border` | `#2f9e44` | Рамка карточки-победителя |
| `--color-surface` | `#f8f9fa` | Фон страницы |
| `--color-card` | `#ffffff` | Фон карточек |
| `--color-text-primary` | `#1a1b1e` | Основной текст |
| `--color-text-secondary` | `#868e96` | Подписи, второстепенный текст |
| `--color-metric-bg` | `#f1f3f5` | Фон MetricCard |

Используется цветовая схема Mantine (`theme.colors`). Все цвета описываются через Mantine-токены: `blue.6`, `orange.7`, `red.7`, `green.7` и т.д. Кастомные CSS-переменные — только для специфических UI-элементов.

### 2.2 Типографика

| Элемент | Font-size | Font-weight | Line-height |
|---|---|---|---|
| H1 (название приложения) | 28px (desktop), 22px (mobile) | 700 | 1.2 |
| H2 (заголовки секций) | 20px | 600 | 1.3 |
| H3 (заголовки карточек) | 16px | 600 | 1.4 |
| Body | 14px | 400 | 1.5 |
| Caption | 12px | 400 | 1.4 |
| MetricCard value | 22px (desktop), 18px (mobile) | 700 | 1.2 |
| MetricCard label | 12px | 400 | 1.4 |

Шрифт: системный стек Mantine (`-apple-system, BlinkMacSystemFont, Segoe UI, ...`).

### 2.3 Скругления и тени

- Карточки (Paper): `radius="md"` (8px)
- MetricCard: `radius="sm"` (4px)
- Кнопки (если есть): `radius="sm"`
- Тень карточек: `shadow="sm"` (Mantine default)
- Sticky-бар: `box-shadow: 0 2px 8px rgba(0,0,0,0.1)`

## 3. Шапка (Header)

### 3.1 Компоновка

**Desktop:** горизонтальная строка. Логотип + название слева, подзаголовок справа (или под названием).  
**Mobile:** вертикальный стэк, всё по центру.

### 3.2 Содержимое

**Логотип:**
- SVG-иконка дома с графиком (16×16px или 24×24px), цвет `blue.6`
- Размещается слева от названия
- Если нет иконки: emoji 🏠 (только если разработчик не создаёт SVG)

**Название:**
- Текст: `Ипотечный стратег`
- Компонент: Mantine `Title order={1}`
- Размер: 28px desktop / 22px mobile
- Цвет: `--color-text-primary`

**Подзаголовок:**
- Текст: `Сравнение стратегий: досрочное погашение vs инвестиции`
- Компонент: Mantine `Text c="dimmed" size="sm"`
- На desktop: под названием (одна строка)
- На mobile: под названием, выровнен по центру

### 3.3 Шапка — разметка (псевдоструктура)

```
[Логотип + Название]      [Подзаголовок (dimmed)]
```

Desktop: `display: flex; align-items: baseline; gap: 16px`  
Либо:
```
[Логотип] [Название]
          [Подзаголовок]
```

На мобильном: всё стэкируется вертикально, `align-items: center`, `text-align: center`.

### 3.4 Фон шапки

Фон: `#ffffff`. Нижняя граница: `border-bottom: 1px solid #e9ecef`. Вертикальный padding: `py: 20px` (desktop), `py: 16px` (mobile).

## 4. Sticky-панель ключевых метрик

> **Дизайнерское решение #1.** Добавлена sticky-панель, которой нет в исходном задании. Обоснование: пользователь постоянно прокручивает страницу между параметрами и графиками — ключевые числа должны быть всегда видны. Подробнее — в секции 13.

### 4.1 Поведение

- Позиция: `position: sticky; top: 0; z-index: 100`
- Появляется: когда пользователь прокрутил ниже шапки (≥ 80px от верха)
- Скрыта изначально (при загрузке страницы шапка видна)
- Анимация появления: `transition: transform 200ms ease` (слайд сверху)
- Фон: `#ffffff` с тенью `box-shadow: 0 2px 8px rgba(0,0,0,0.1)`

### 4.2 Содержимое (5 чисел в строку)

| Позиция | Метка | Значение | Цвет |
|---|---|---|---|
| 1 | Кредит | loanAmount ₽ | текст |
| 2 | Мин. платёж | PMT ₽/мес | текст |
| 3 | Победитель | "Стратегия А" / "Стратегия Б" | оранжевый / синий |
| 4 | Разница | ΔnetWorth ₽ | зелёный если > 0 |
| 5 | Точка безопасности | "Мес. N" или "Не достигнута" | зелёный / красный |

### 4.3 Разметка sticky-бара

```
[Кредит: 5,5 млн ₽] | [Платёж: 39 404 ₽/мес] | [Победитель: Стратегия Б] | [Преимущество: +1,2 млн ₽] | [Безопасность: мес. 28]
```

На mobile: sticky-бар скрыт (слишком узко). Вместо него — краткий баннер под шапкой (только победитель + разница).

### 4.4 Компонент Mantine

`Paper` с `p="sm"`, без `radius` (flat) или с `radius={0}`. Внутри: `Group justify="space-around"`.

## 5. Секция: Параметры ипотеки

### 5.1 Обёртка

Компонент: Mantine `Paper` с `p="lg"` и `shadow="sm"` и `radius="md"`.

Заголовок секции: `Text fw={600} size="lg"` — `Параметры ипотеки`.  
Под заголовком: `Divider mb="md"`.

### 5.2 Двухколоночная сетка

Компонент: Mantine `SimpleGrid cols={{ base: 1, sm: 2 }}` с `spacing="xl"`.

### 5.3 Левая колонка — параметры кредита

#### 5.3.1 Слайдер «Стоимость квартиры»

- Компонент: Mantine `Slider`
- Метка (label над слайдером): `Стоимость квартиры`
- Диапазон: min=1_000_000, max=20_000_000, step=100_000
- Значение по умолчанию: `6_500_000`
- Формат значения над ползунком: `"X млн ₽"` (если ≥ 1M) или `"X ₽"` (если < 1M)  
  Пример: `6,5 млн ₽`
- Метки оси (marks): `[{value: 1_000_000, label: '1M'}, {value: 5_000_000, label: '5M'}, {value: 10_000_000, label: '10M'}, {value: 20_000_000, label: '20M'}]`
- `labelAlwaysOn`: false (показывать только при drag)

#### 5.3.2 Слайдер «Первоначальный взнос»

- Компонент: Mantine `Slider`
- Метка: `Первоначальный взнос`
- Диапазон: min=10, max=50, step=1
- Значение по умолчанию: `15`
- Формат значения над ползунком: `"X%"`
- Метки оси: `[{value: 10, label: '10%'}, {value: 20, label: '20%'}, {value: 30, label: '30%'}, {value: 50, label: '50%'}]`
- **Под слайдером** — строка `Text size="sm" c="dimmed"`:  
  `Сумма взноса: {downPaymentAmount} ₽`  
  Пример: `Сумма взноса: 975 000 ₽`
  - Обновляется мгновенно при движении слайдера

#### 5.3.3 Блок «Сумма кредита» (только чтение)

- Компонент: Mantine `Paper` с `bg="blue.0"` и `p="sm"` и `radius="sm"`
- Внутри: 
  - `Text size="xs" c="blue.7" fw={500}` → `СУММА КРЕДИТА`
  - `Text size="xl" fw={700} c="blue.7"` → `{loanAmount} ₽`
  - Пример: `5 525 000 ₽`
- Если loanAmount = 0: вместо суммы показывать `Кредит не требуется` + иконка ✓ зелёным
- Этот блок визуально выделен на фоне слайдеров — синеватый фон, чёткая рамка

#### 5.3.4 Слайдер «Льготная ставка ИТ-ипотеки»

- Компонент: Mantine `Slider`
- Метка: `Льготная ставка ИТ-ипотеки`
- Диапазон: min=3, max=8, step=0.1
- Значение по умолчанию: `6`
- Формат: `"X%"`
- Метки оси: `[{value: 3, label: '3%'}, {value: 5, label: '5%'}, {value: 6.5, label: '6.5%'}, {value: 8, label: '8%'}]`
- Цвет слайдера: `green` (льготная ставка — позитивный параметр)

#### 5.3.5 Слайдер «Срок ипотеки»

- Компонент: Mantine `Slider`
- Метка: `Срок ипотеки`
- Диапазон: min=5, max=30, step=1
- Значение по умолчанию: `20`
- Формат: `"X лет"`
- Метки оси: `[{value: 5, label: '5 лет'}, {value: 10, label: '10'}, {value: 15, label: '15'}, {value: 20, label: '20'}, {value: 30, label: '30 лет'}]`

### 5.4 Правая колонка — инвестиционные параметры

#### 5.4.1 Слайдер «Свободные деньги в месяц»

- Компонент: Mantine `Slider`
- Метка: `Свободные деньги в месяц`
- Подпись под меткой (`Text size="xs" c="dimmed"`): `Сумма поверх обязательного платежа, которую вы готовы направить на ипотечные цели`
- Диапазон: min=20_000, max=500_000, step=5_000
- Значение по умолчанию: `100_000`
- Формат: `"X ₽/мес"`
- Метки оси: `[{value: 20_000, label: '20к'}, {value: 100_000, label: '100к'}, {value: 250_000, label: '250к'}, {value: 500_000, label: '500к'}]`
- **Предупреждение** (появляется если freeMonthly < PMT):
  - Под слайдером: Mantine `Alert` c `color="yellow"` и `icon` (предупреждение)
  - Текст: `Свободные средства меньше минимального платежа (${PMT} ₽). Обе стратегии идентичны.`

#### 5.4.2 Слайдер «Доходность депозита/облигаций»

- Компонент: Mantine `Slider`
- Метка: `Доходность депозита / облигаций`
- Диапазон: min=6, max=25, step=0.5
- Значение по умолчанию: `16`
- Формат: `"X% годовых"`
- Метки оси: `[{value: 6, label: '6%'}, {value: 10, label: '10%'}, {value: 16, label: '16%'}, {value: 25, label: '25%'}]`
- Цветная точка-индикатор рядом с меткой: зелёная если depositRate > itRate + 3%, жёлтая если разница 0–3%, красная если depositRate < itRate. Реализация: цветной маленький `Badge` без текста, или точка `●` через CSS.

#### 5.4.3 Слайдер «Горизонт сравнения»

- Компонент: Mantine `Slider`
- Метка: `Горизонт сравнения`
- Диапазон: min=3, max=25, step=1
- Значение по умолчанию: `10`
- Формат: `"X лет"`
- Метки оси: `[{value: 3, label: '3 г.'}, {value: 5, label: '5'}, {value: 10, label: '10'}, {value: 15, label: '15'}, {value: 25, label: '25 лет'}]`
- **Ограничение:** не должен превышать срок ипотеки. Если mortgageTerm < horizonYears, слайдер автоматически ограничивается значением mortgageTerm и показывает `Text size="xs" c="dimmed"`: `Ограничено сроком ипотеки`.

#### 5.4.4 NumberInput «Зарплата до налогов»

- Компонент: Mantine `NumberInput`
- Метка: `Зарплата до налогов`
- Подпись: `Для расчёта налоговых вычетов. 0 — не учитывать.`
- Placeholder: `0`
- Suffix: ` ₽/мес`
- Min: 0, Max: 10_000_000, step: 1_000
- Значение по умолчанию: `0`
- Правая иконка (rightSection): иконка информации `ℹ` с Mantine `Tooltip`:  
  `Ставка НДФЛ зависит от годового дохода. До 200 000 ₽/мес — 13%, до ~417 000 ₽/мес — 15%, выше — 18%.`
- Форматирование: разделитель тысяч (пробел или запятая)
- При значении > 0: под полем появляется `Text size="xs" c="dimmed"`:  
  `Ставка НДФЛ: {ndflRate * 100}% → Максимальный вычет: {totalDeduction} ₽`

### 5.5 Метрические карточки (4 штуки)

**Расположение:** под двухколоночной сеткой, на всю ширину. `SimpleGrid cols={{ base: 2, sm: 4 }}` с `spacing="md"`.

#### MetricCard — компонент

Каждая карточка:
- Компонент: Mantine `Paper` с `bg="gray.0"` и `p="md"` и `radius="md"`
- Внутри:
  - `Text size="xs" c="dimmed" tt="uppercase" fw={500}` → метка (лейбл)
  - `Text size="xl" fw={700}` → значение
  - Опционально: иконка или акцентный цвет значения

#### MetricCard #1 — «Минимальный платёж»

- Метка: `Минимальный платёж`
- Значение: `{PMT} ₽/мес`
- Пример: `39 404 ₽/мес`
- Цвет значения: `blue.7`
- Иконка слева от метки: 📅 (или Mantine иконка календаря из Tabler Icons)

#### MetricCard #2 — «Итоговая переплата»

- Метка: `Переплата по плану`
- Значение: `{totalInterest} ₽`
- Пример: `4 056 960 ₽`
- Цвет значения: `red.7`
- Подпись под значением (`Text size="xs" c="dimmed"`): `без досрочного погашения`

#### MetricCard #3 — «Ставка НДФЛ»

- Метка: `Ставка НДФЛ`
- Значение: `{ndflRate * 100}%`
- Если salary = 0: значение `Не указана`, цвет `gray.5`
- Если salary > 0: цвет `orange.7`

#### MetricCard #4 — «Суммарный вычет»

- Метка: `Налоговые вычеты`
- Значение: `{totalDeduction} ₽` (если salary > 0)
- Если salary = 0: значение `Укажите зарплату`, цвет `gray.5`
- Если salary > 0: цвет `green.7`
- Tooltip на карточке (Mantine Tooltip): `Имущественный вычет ${propertyDeductionTotal} ₽ + Вычет по процентам ${interestDeductionTotal} ₽`

## 6. Секция: Сценарий слёта

### 6.1 Обёртка

Компонент: Mantine `Paper` с `p="lg"` и `shadow="sm"` и `radius="md"`.  
**Стилевой акцент:** `style={{ borderLeft: '4px solid var(--mantine-color-orange-6)' }}`.

### 6.2 Заголовок секции

- `Group gap="xs"`:
  - Иконка предупреждения: Mantine `ThemeIcon` с `color="orange"` и `variant="light"` + иконка `IconAlertTriangle` (Tabler Icons)
  - `Text fw={600} size="lg"` → `Сценарий слёта с ИТ-ипотеки`
- Под заголовком: `Text size="sm" c="dimmed"`:  
  `Моделирование ситуации, когда льготная программа прекращается и банк переводит кредит на рыночные условия.`
- `Divider mb="md" mt="xs"`

### 6.3 Слайдер «Месяц слёта»

- Компонент: Mantine `Slider`
- Метка: `Месяц слёта`
- Диапазон: min=0, max=mortgageTerm*12, step=1
- Значение по умолчанию: `36`
- Формат значения над ползунком:
  - `0` → `Слёт не происходит`
  - `N > 0` → `Месяц N`
- Метки оси: через каждые 12 месяцев с лейблом `Y год` (Y = month / 12)  
  Пример: `[{value: 0, label: 'Сейчас'}, {value: 12, label: '1 г.'}, {value: 24, label: '2 г.'}, ..., {value: mortgageTerm*12, label: 'Конец'}]`
- Цвет слайдера: `orange`

**Динамическая подпись под слайдером:**
- `Text size="sm" c="dimmed"` (обновляется при движении)
- Если slipMonth = 0:  
  `Слёт не происходит — рассчитывается базовый сценарий`
- Если slipMonth > 0:  
  `Через {floor(slipMonth/12)} лет {slipMonth%12} месяцев после начала ипотеки`  
  Если slipMonth кратно 12: `Через {slipMonth/12} лет`  
  Если slipMonth < 12: `Через {slipMonth} месяцев`

### 6.4 Слайдер «Ключевая ставка при слёте»

- Компонент: Mantine `Slider`
- Метка: `Ключевая ставка при слёте`
- Диапазон: min=8, max=25, step=0.5
- Значение по умолчанию: `16`
- Формат: `"X%"`
- Метки оси: `[{value: 8, label: '8%'}, {value: 13, label: '13%'}, {value: 16, label: '16%'}, {value: 21, label: '21%'}, {value: 25, label: '25%'}]`
- Цвет слайдера: `red`

### 6.5 Слайдер «Дисконт банка»

- Компонент: Mantine `Slider`
- Метка: `Group` с `Text "Дисконт банка"` + Mantine `Tooltip` на иконке `ℹ`:
  - Tooltip текст: `Дисконт к ключевой ставке, который банк применяет при смене условий. Обычно 0,5%. Итоговая ставка = ключевая − дисконт + 1,5%`
  - `Tooltip.multiline`, `width={280}`
- Диапазон: min=0, max=2, step=0.1
- Значение по умолчанию: `0.5`
- Формат: `"X%"`

## 11. Поведение и взаимодействие

### 11.1 Реактивность вычислений

- **Нет кнопки «Рассчитать»**. Все изменения параметров немедленно запускают пересчёт.
- Debounce на слайдерах: **50ms** (предотвращает избыточные пересчёты при быстром движении)
- NumberInput (зарплата): debounce **300ms**
- После debounce — синхронный пересчёт всех данных (нет асинхронных запросов)

### 11.2 Слайдеры Mantine

- `labelAlwaysOn={false}` — пузырёк со значением появляется только при drag
- `label` пропс — функция форматирования: `label={(value) => formatSliderValue(value)}`
- При focus на слайдере: пузырёк появляется
- При release: пузырёк скрывается через 1.5 сек (стандартное поведение Mantine)

### 11.3 Взаимозависимости параметров

**slipMonth ↔ mortgageTerm:**
- Если пользователь уменьшает срок ипотеки так, что slipMonth > mortgageTerm × 12:
  - slipMonth автоматически ограничивается до mortgageTerm × 12
  - Под слайдером появляется `Text size="xs" c="orange.7"`: `Месяц слёта скорректирован до конца ипотеки`

**horizonYears ↔ mortgageTerm:**
- Если horizonYears > mortgageTerm: автоматически ограничивается
- Под слайдером: `Text size="xs" c="dimmed"`: `Ограничено сроком ипотеки`

**loanAmount = 0:**
- Секция сценария слёта скрывается (нет ипотеки — нет слёта)
- Секция сравнения стратегий показывает: `Alert color="blue"`: `Первоначальный взнос покрывает полную стоимость. Ипотека не требуется.`

**freeMonthly < PMT:**
- Предупреждение в правой колонке секции 2
- Карточки стратегий показывают одинаковые значения с пояснением

### 11.4 Скролл и навигация

Нет якорных ссылок в шапке (пользователь не просил).  
Плавный скролл стандартный (`scroll-behavior: smooth` если нужен якорный переход из будущих фич).

### 11.5 Пустые состояния

Все поля имеют значения по умолчанию. Пустое состояние (все нули) не достигается при нормальном использовании слайдеров.

## 13. Дизайнерские решения

### ДР-1: Sticky-панель с ключевыми метриками

**Проблема:** При прокрутке от параметров к графикам пользователь теряет контекст ключевых чисел (какой кредит, какой платёж, кто побеждает).

**Решение:** Добавлена sticky-панель, появляющаяся после прокрутки шапки. Показывает 5 критических метрик: сумму кредита, минимальный платёж, победителя, разницу и точку безопасности.

**Компромисс:** на mobile скрывается (слишком узко). Вместо неё — краткий статичный баннер победителя под шапкой.

---

### ДР-2: Accordion вместо Tabs в секции советов

**Проблема:** `Tabs` скрывает все вкладки кроме одной. На mobile пользователь должен переключаться и не видит общей картины. `Accordion` позволяет открыть несколько пунктов одновременно.

**Решение:** Mantine `Accordion variant="separated"` с `defaultValue="recommendation"`.

**Преимущества:**
- Пользователь может читать советы и методологию одновременно
- Лучше работает на mobile (вертикальная прокрутка естественнее вкладок)
- SEO-friendly (контент не скрыт в display:none)
- Налоговый вычет скрывается условно (salary = 0) — с Accordion это проще

---

### ДР-3: Группировка графиков 1+2 рядом на wide desktop

**Проблема:** На широких экранах (≥ 1200px) три последовательных графика высотой 300px занимают огромную вертикаль. Пользователь прокручивает, не видя связи между ними.

**Решение:** На `lg` breakpoint (≥ 1200px) — графики 1 и 2 размещаются рядом в 2 колонки. График 3 — полная ширина под ними.

**Реализация:** `SimpleGrid cols={{ base: 1, lg: 2 }}` для первых двух, отдельный блок для третьего.

**Предостережение для разработчика:** на lg нужно убедиться, что `ResponsiveContainer` корректно подхватывает ширину внутри grid-ячейки.

---

### ДР-4: Цветовой индикатор разрыва ставок у слайдера depositRate

**Проблема:** Пользователь не всегда понимает, насколько значимо соотношение depositRate и itRate. Нужна мгновенная обратная связь.

**Решение:** Маленькая цветная точка/Badge рядом с меткой слайдера доходности:
- Зелёная: `depositRate > itRate + 3` — инвестиции явно выгоднее
- Жёлтая: `0 ≤ depositRate − itRate ≤ 3` — паритет
- Красная: `depositRate < itRate` — досрочное погашение выгоднее

---

### ДР-5: Вынесение «Суммы кредита» в отдельный акцентный блок

**Проблема:** «Сумма кредита» — производный параметр от стоимости и взноса. В исходном задании он указан как `Text` только для чтения, но это самое важное число для всех расчётов.

**Решение:** Оформлен как `Paper bg="blue.0"` с крупным шрифтом и чётким лейблом. Визуально выделяется в потоке слайдеров.

---

### ДР-6: Удаление вкладки «Налоговые вычеты» при salary = 0

**Проблема:** В Tabs пустая вкладка с сообщением «укажите зарплату» — плохой UX. В Accordion — ещё хуже.

**Решение:** Accordion.Item `"tax"` не рендерится вовсе при `salary === 0`. При появлении зарплаты — пункт добавляется с анимацией (Mantine Transition).

---

### ДР-7: Подпись под слайдером первоначального взноса

**Проблема:** Процент взноса абстрактен. Пользователь думает в рублях.

**Решение:** Под слайдером — динамическая строка `Сумма взноса: X ₽`, обновляющаяся в реальном времени. Дополнительно — динамическая сумма кредита в акцентном блоке ниже.

---

### ДР-8: Ограничение горизонта сравнения сроком ипотеки

**Проблема:** Если горизонт > срока ипотеки, последние точки на графике бессмысленны (долг = 0, но анализ продолжается). Это корректно математически, но вводит в заблуждение.

**Решение:** Слайдер горизонта автоматически ограничивается значением mortgageTerm. При этом пользователь видит пояснение. Это предотвращает путаницу без ограничения функциональности.

---

### ДР-9: Тултипы всех графиков отображают PMT как опорное значение

**Проблема:** На графике 3 пользователь хочет знать — «а это выше или ниже моего льготного платежа?» без поиска глазами.

**Решение:** Во всех тултипах Recharts добавлена строка `Льготный PMT: X ₽` для контекста. Дополнительно — горизонтальная зелёная линия PMT на графике 3.

## Приложение: Спецификация форматирования чисел

### Функция `formatRub(value)`

| Значение | Отображение |
|---|---|
| `< 1000` | `123 ₽` |
| `1 000 – 999 999` | `39 404 ₽` (пробел-разделитель) |
| `1 000 000 – 999 999 999` | `5,5 млн ₽` или `5 031 775 ₽` (зависит от контекста) |
| В MetricCard крупном | Всегда полное число с разделителями |
| В подписях и тултипах | Сокращённое (млн) |

### Функция `formatSliderValue(value, type)`

| type | Формат |
|---|---|
| `apartment_price` | `6,5 млн ₽` |
| `down_payment` | `15%` |
| `rate` | `6,0%` |
| `years` | `20 лет` |
| `money_per_month` | `100 000 ₽/мес` |
| `rate_annual` | `16% годовых` |
| `month` | `Мес. 36` или `Слёт не происходит` |

### Функция `formatMonthsToYearsMonths(months)`

`36` → `3 года`  
`38` → `3 года 2 месяца`  
`5` → `5 месяцев`  
`12` → `1 год`

Падежи: 1 год / 2 года / 5 лет; 1 месяц / 2 месяца / 5 месяцев.
